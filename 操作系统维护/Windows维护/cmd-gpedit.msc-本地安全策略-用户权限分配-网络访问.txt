从网络访问此计算机

此策略允许网络上的其他用户连接到此计算机，
并且是许多网络协议所必需的，
这些协议包括
基于服务器消息块 (SMB) 的协议、
网络基本输入/输出系统 (NetBIOS)、
通用 Internet 文件系统 (CIFS)和
组件对象模型 (COM+)。 
在SCM配置用户权限时输入逗号分隔帐户列表。
可以是本地帐户或活动目录帐户，可以是组帐户、用户帐户或者计算机帐户。

将计算机连接到网络上的用户可以访问目标计算机中其拥有权限的资源。
例如，用户连接共享的打印机和文件夹需要从网络访问此计算机用户权限。
如果将此用户权限分配给每个用户组，那么任何组员都可以阅读共享文件夹中的文件。
然而，这个情况不太可能出现在装有Service Pack 1 (SP1)的新安装的Windows Server® 2003，
因为Windows Server 2003中的默认分享和NTFS权限不包括每个用户组。
这个漏洞可能会使由Windows NT® 4.0 or Windows 2000升级而来的计算机拥有更高风险，
因为这些操作系统的默认权限不像Windows Server 2003限制多多。

限制从网络访问此计算机用户权限只分配给那些需要访问服务器的用户。
例如，如果您将此策略设置配置到管理员和用户组，
若本地用户组包括域用户组成员，登录到网域的用户将可以访问本网域服务器中共享的资源。

计算机配置\Windows设置\安全设置\本地策略\用户权限分配

如果您在网域控制台对所有用户取消从网络访问此计算机用户权限，
则没有人可以登录到此网域或者使用网络资源。
如果取消成员服务器上的此用户权限，用户将不能通过网络连接到那些服务器。
成功完成IPsec连接要求最初的机器有此权限，
因此微软建议将其分配给用户组。
如果您安装了可选组件如ASP.NET或者是互联网信息服务(IIS)，
您可能需要将此用户权限分配给这些组件所需要的附加帐户。
重要的是要核实此用户权限已分配给已授权用户需要访问网络的计算机。

用户权限用于控制所依据的用户可以登录到系统的方法。 
用户权限在本地计算机级别应用，并允许用户在计算机上或在域中执行任务。 
用户权限包括登录权限和权限。 
登录权限控制有权登录到计算机并如何才能登录。 
用户权限控制对计算机和域资源的访问并且它们可重写已对特定对象中设置的权限。
